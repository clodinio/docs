## Requesting QDA-Compatible Formatting

When placing your order, specify your QDA software in the order notes field. We will format the transcript accordingly. Available formats:

### NVivo Formats

**NVivo Synchronized** Includes timestamps at every speaker turn. Allows NVivo to sync the transcript with your audio or video file so you can click a passage and jump directly to the corresponding moment in the recording. Recommended for focus groups and interviews where tone and delivery are analytically relevant.

**NVivo Headings** Transcript organized by interview questions or thematic sections. NVivo can auto-code by question or speaker using the heading structure, significantly reducing manual node creation. Recommended for structured interviews with predefined topics.

**NVivo Basic** Clean text import. Compatible with all NVivo versions. No timestamps or heading structure. Suitable for manual coding or older NVivo installations.

### ATLAS.ti, MAXQDA, Dedoose

These tools import standard DOCX or plain text files. Specify your preferred software and we will format speaker labels, paragraph breaks, and timestamp placement to work with each tool's import requirements.

## Speaker Label Conventions

Consistent speaker labels are essential for auto-coding and node creation. Common conventions:

**Research interviews:**

- `Interviewer:` and `Participant:` or `P1:`, `P2:` for multiple participants

**Focus groups:**

- `Moderator:` and `R1:`, `R2:` through `R8:` (or participant names if provided)

**Legal proceedings:**

- `Examining Counsel:`, `Witness:`, `Opposing Counsel:`, `Judge:`

Specify your preferred labeling system in the order notes and we will apply it consistently throughout.

## Importing into NVivo

1. Open NVivo and create or open your project
2. Go to **Data \> Files \> Import** and select your transcript file (.docx or .txt)
3. Add metadata (participant name, interview date, project)
4. Ensure paragraph formatting is clean; NVivo uses each paragraph as a coding unit

For NVivo Synchronized transcripts, link your media file to the transcript after import to enable media playback.

## Importing into ATLAS.ti

1. Open your ATLAS.ti project
2. Go to **Documents \> Add Documents** and select your transcript file
3. Assign the document to the relevant document group
4. Begin coding directly in the document viewer

## Smart Insights Coding CSV

Smart Insights generates a Coding CSV formatted for direct import into NVivo, ATLAS.ti, MAXQDA, or Dedoose. The CSV contains your thematic structure, speaker data, and coded segments in a format each tool recognizes.

To use: run Smart Insights on your completed transcript, download the Coding CSV, and import it into your QDA software. This gives you a starting framework to refine rather than building your coding structure from scratch.

## Best Practices

**Request timestamps.** Even if you plan to code manually, timestamps allow you to return to the original recording for verification, double-check ambiguous passages, and cross-reference quotes with audio.

**Provide speaker names upfront.** Give us speaker names or roles in your order notes. Changing speaker labels after delivery is time-consuming. Getting them right from the start is worth the extra minute.

**Use consistent formatting across your project.** If you are analyzing 20 interviews, all transcripts should use the same speaker label format, timestamp style, and paragraph structure. Order them all the same way.

**Do a light proofread.** Even with high accuracy, a quick pass against the audio before you begin coding helps. Catching a misheard technical term before it is coded into 15 nodes saves significant rework.

**Use Smart Insights as a first pass, not a final answer.** The thematic structure Smart Insights generates gives you a starting framework and a quotes bank. Your interpretive analysis is what makes the work meaningful. Do not skip the reading.
